# 创作管线

> 从构思到完稿，novel-harness 的完整创作流程。

---

## 单章完整流水线

```
用户确认"写第N章"
  │
  ▼
① 上下文 Agent — 打包当前上下文
   ├─ 主角状态（等级/属性/位置）
   ├─ 出场角色
   ├─ 活跃伏笔（标记超期）
   ├─ 风格参考（最近3章特征）
   └─ 约束提醒（⚠ 不能违反的规则）
  │
  ▼
② 写作 Agent — 生成正文
   ├─ 接收上下文包 + 大纲
   ├─ 写 2000-3000 字
   └─ 遵守 human-linguistics 语感规则
  │
  ▼
③ 上下文 Agent — 写作后归档
   ├─ 更新主角状态
   ├─ 更新伏笔登记表
   ├─ 更新事件索引
   ├─ 生成章节摘要
   └─ git commit
  │
  ▼
④ 审稿 Agent — 质量检查
   ├─ 语感检查（human-linguistics）
   ├─ 情节检查（plot-review）
   ├─ 节奏检查（rhythm-review）
   └─ 题材检查（game-datafied）
  │
  ▼
⑤ 总编 Agent — 汇总呈现
   ├─ 展示修改建议
   └─ 等待用户确认
```

---

## 200 万字创作周期

| 阶段 | 章节范围 | 核心任务 |
|------|---------|---------|
| 第一卷 | 第 1-30 章 | 建立世界观 + 主角初始成长 |
| 第二卷 | 第 31-80 章 | 扩大格局 + 引入势力冲突 |
| 第三卷 | 第 81-150 章 | 深化主线 + 伏笔集中回收 |
| 第四卷 | 第 151-250 章 | 世界观揭秘 + 格局升级 |
| 第五卷+ | 第 251 章+ | 终极冲突 + 收束全部线索 |

每卷结束后由规划 Agent + 上下文 Agent 做卷总结和下一卷大纲调整。

---

## 状态与记忆系统

### 状态文件（由上下文 Agent 自动维护，禁止手动修改）

| 文件 | 内容 | 更新时机 | 读取者 |
|------|------|---------|--------|
| `状态/主角.md` | 等级/属性/位置/任务/装备 | 每章完成后 | 写作/规划/审稿 Agent |
| `状态/角色/王磊.md` | 配角状态/关系/已知信息 | 该角色出场后 | 写作/审稿 Agent |
| `状态/伏笔登记表.md` | 编号/内容/埋下章/预计回收章 | 每章完成后 | 规划 Agent |

### 记忆文件（由上下文 Agent 自动维护）

| 文件 | 内容 | 用途 |
|------|------|------|
| `记忆/事件索引.md` | 重大事件按章索引 | 快速定位历史事件 |
| `记忆/章节摘要/第N章.md` | 200 字章节概要 | 上下文包素材 |
| `记忆/风格参考.md` | 最近 5 章写作特征 | 风格一致性保持 |

### 上下文包（每次写作前临时生成）

```
━━ 上下文包 ━━
── 1. 当前任务 ──    ← 本章要写什么
── 2. 主角状态 ──    ← 主角精确的当前状态
── 3. 出场角色 ──    ← 本章涉及的角色
── 4. 活跃伏笔 ──    ← 未回收的伏笔（超期标记）
── 5. 风格参考 ──    ← 最近3章节奏特征
── 6. 约束提醒 ⚠    ← 不能违反的硬约束
```

### 生命周期

```
写作前                  写作中                  写作后
  │                      │                      │
  ▼                      ▼                      ▼
读取状态文件    →   Agent 参考状态    →   更新状态文件
读取记忆文件    →   Agent 参考记忆    →   更新记忆文件
打包上下文包    →   注入写作 Agent    →   git commit
```
