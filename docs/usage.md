# 使用指南

> 从零开始使用 novel-harness 进行长篇创作。

---

## 开始写作

### 第一步：创建小说项目

在 Claude 中调用 novel-harness，Agent 会引导你完成：

```
你：帮我创建新项目
Agent：好的，几个问题：
  1. 书名？
  2. 题材？（数据化降临 / 末世小黑屋 / 其他）
  3. 主角叫什么？
  4. 一句话世界观？

确认后生成：
  projects/{书名}/
  ├── 正文/
  ├── 状态/（由上下文 Agent 自动维护）
  └── 记忆/（由上下文 Agent 自动维护）
```

### 第二步：日常写作

```
你：帮我写第二章
→ 总编 Agent 协调全流程：上下文打包 → 写作 → 归档 → 审稿

你：帮我审稿
→ 委派审稿 Agent，检查语感/设定/情节/节奏

你：没灵感，后面怎么写
→ 委派规划 Agent，提供 3-5 个剧情方向
```

## RAG 知识检索层

RAG（Retrieval-Augmented Generation）层是 Agent 的"知识外脑"，为写作提供即时知识查询。

### 解决的问题

- **Agent 遗忘**：长篇写作中，Agent 容易忘记之前用过的规则和知识
- **知识分散**：规则、参考、模板分散在各目录，Agent 难以快速定位
- **上下文窗口限制**：无法把所有参考文档塞进 prompt，需要精准检索

### 安装

```bash
pip install -r rag/requirements.txt
python rag/scripts/build_index.py
```

首次运行会自动下载 sentence-transformers 多语言模型（~100MB）。安装完成后，Agent 会自动通过 HTTP API 调用 RAG 服务。

### 能力展示

| 查询 | 返回的知识 |
|------|-----------|
| "这段太像 AI 写的" | 去AI味最小修改指南 |
| "这个大纲能展开吗" | 大纲质量评估清单 |
| "写之前该准备什么" | 章节写前准备清单 |
| "全民求生选哪条路线" | 子题材规则入口 |
| "角色状态有没有矛盾" | 状态一致性补充清单 |

### 什么时候需要重建索引

- 首次部署后
- 修改了 `.harness/skills/` 中的规则或参考文档
- 新增了项目模板

```bash
python rag/scripts/build_index.py
```

详细操作手册见 `rag/OPERATIONS.md`。

## 用户分层

系统识别用户的写作经验等级，按不同方式交付反馈：

| 等级 | 特征 | 反馈方式 |
|------|------|---------|
| **Lv.1 零基础** | 第一次写小说 | 先夸再批、每次 ≤3 条问题、直接给修改全文 |
| **Lv.2 老书虫** | 审美在线但手生 | 直言问题、给方向+1 处示范、5-7 条分类 |
| **Lv.3 资深作者** | 有自己的判断标准 | 只标记问题、由作者决策、同行协作姿态 |

## 项目自定义

每个小说项目有自己的创作约束模板，位于 `.harness/projects/`。

### 预置模板

| 模板 | 适用题材 | 特点 |
|------|---------|------|
| `模板-数据化降临.md` | 游戏降临 / 副本流 | 数值驱动型成长 |
| `模板-末世小黑屋.md` | 末世 / 安全屋 / 副本 | 非数值驱动、防膨胀、同质化防控 |

### 自定义方法

复制一个模板，修改以下参数即可创建新题材模板：

```
成长曲线：数值驱动 / 非数值驱动 / 混合型
数值膨胀控制：严格 / 一般 / 不控制
同质化防控：开 / 关
阶段天花板：开 / 关
装备系统：替换制 / 改造制
```

## Git 工作流

### 初始化

```bash
# 在项目根目录执行一次
git init
git add .
git commit -m "init: 项目初始化"
```

### 每章完成后的本地记录

`projects/` 下的具体创作内容默认不提交到远程。若你只在本地管理小说版本，可以继续用 Git 记录：

```bash
git add projects/{项目名}/正文/  projects/{项目名}/状态/  projects/{项目名}/记忆/
git commit -m "Ch{N}: {章节标题} — {一句话钩子}"
```

### 关键操作

```bash
# 回溯某章时的主角状态
git show Ch120:projects/{项目名}/状态/主角.md

# 查看两章之间的属性变化
git diff Ch120..Ch130 projects/{项目名}/状态/主角.md

# 实验性分支（测试新剧情方向）
git branch 实验-分支名
git checkout 实验-分支名
# 写几章试试...不满意就切回主线
git checkout main

# 审稿对比（看改了什么）
git diff HEAD~1 projects/{项目名}/正文/第80章.md
```

## 与普通 Skill 的区别

| 维度 | 普通 Skill | novel-harness |
|------|-----------|---------------|
| **角色** | 工具，你调它 | **Agent 系统，它协调你** |
| **记忆** | 每次调用零上下文 | **持久化状态 + 伏笔追踪 + 事件索引** |
| **规模** | 单次对话 | **100-200 万字连载** |
| **架构** | 单文件 SKILL.md | **L0-L3 四层 + 4 专业 Agent** |
| **写作** | 不支持 | **规划 → 上下文注入 → 写作 → 审查** |
| **自我进化** | 无 | **反馈追踪 + 3 次误判触发规则更新** |
| **Git** | 无 | **每章 commit + 回溯 + 分支实验** |
| **容量** | 几千字 | **200 万字** |
