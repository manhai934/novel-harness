# 使用指南

> 项目自定义、用户分层、Git 工作流等进阶操作。

---

## 用户分层

系统识别用户的写作经验等级，按不同方式交付反馈：

| 等级 | 特征 | 反馈方式 |
|------|------|---------|
| **Lv.1 零基础** | 第一次写小说 | 先夸再批、每次 ≤3 条问题、直接给修改全文 |
| **Lv.2 老书虫** | 审美在线但手生 | 直言问题、给方向+1 处示范、5-7 条分类 |
| **Lv.3 资深作者** | 有自己的判断标准 | 只标记问题、由作者决策、同行协作姿态 |

## 项目自定义

每个小说项目有自己的创作约束模板，位于 `.harness/project-templates/`。

### 预置模板

| 模板 | 适用题材 | 特点 |
|------|---------|------|
| `模板-数据化降临.md` | 游戏降临 / 副本流 | 数值驱动型成长 |
| `模板-末世小黑屋.md` | 末世 / 安全屋 / 副本 | 非数值驱动、防膨胀、同质化防控 |

### 自定义方法

复制一个模板，修改以下参数即可创建新题材模板：

```
成长曲线：能力成长 / 关系推进 / 资源经营 / 线索追查 / 混合型
能力/资源膨胀控制：严格 / 一般 / 不控制
同质化防控：开 / 关
阶段天花板：开 / 关
核心道具或资源机制：替换制 / 改造制 / 积累制 / 消耗制
```

## Git 工作流

### 初始化

```bash
# 在项目根目录执行一次
git init
git add .
git commit -m "init: 项目初始化"
```

### 每章完成后的本地记录

`projects/` 是本地创作目录，默认被仓库忽略，不提交到远程。开始创作时，Agent 会使用或创建 `projects/{项目名}/` 存放正文、设定、状态和记忆。

若你确实想给某本小说单独做版本管理，更建议在 `projects/{项目名}/` 内单独初始化一个 Git 仓库；如果一定要用当前仓库记录，需要手动 `git add -f`：

```bash
git add -f projects/{项目名}/正文/  projects/{项目名}/状态/  projects/{项目名}/记忆/
git commit -m "Ch{N}: {章节标题} — {一句话钩子}"
```

### 关键操作

```bash
# 回溯某章时的主角状态
git show Ch120:projects/{项目名}/状态/主角.md

# 查看两章之间的状态变化
git diff Ch120..Ch130 projects/{项目名}/状态/主角.md

# 实验性分支（测试新剧情方向）
git branch 实验-分支名
git checkout 实验-分支名
# 写几章试试...不满意就切回主线
git checkout main

# 审稿对比（看改了什么）
git diff HEAD~1 projects/{项目名}/正文/第80章.md
```

## 与普通 Skill 的区别

| 维度 | 普通 Skill | novel-harness |
|------|-----------|---------------|
| **角色** | 工具，你调它 | **Agent 系统，它协调你** |
| **记忆** | 每次调用零上下文 | **持久化状态 + 伏笔追踪 + 事件索引** |
| **规模** | 单次对话 | **100-200 万字连载** |
| **架构** | 单文件 Skill | **L0-L3 四层 + 总编 Agent + 4 专业 Agent** |
| **写作** | 不支持 | **规划 → 上下文注入 → 写作 → 审查** |
| **自我进化** | 无 | **反馈追踪 + 3 次误判触发规则更新** |
| **Git** | 无 | **每章 commit + 回溯 + 分支实验** |
| **容量** | 几千字 | **200 万字** |
