# Agent 体系

> 系统有 4 个专业 Agent，由总编 Agent 统一协调。每个 Agent 有明确的职责边界。

---

## 总编 Agent（大管家）

**职责**：理解需求、分派任务、协调流程、质量裁定

| 你说 | 总编做什么 |
|------|-----------|
| "帮我审稿" | 派审稿 Agent |
| "没灵感" | 派规划 Agent |
| "帮我写一章" | 先派上下文 Agent 打包 → 再派写作 Agent |
| "一条龙" | 规划 → 上下文 → 写作 → 审稿，全流程协调 |

**不做**：不直接审查、不直接写、不自己构思。

---

## 上下文 Agent（记忆中枢）

**职责**：主角状态追踪、伏笔管理、事件索引、上下文打包

### 写作前

打包当前状态 → 注入写作 Agent。上下文包包含：

```
━━ 上下文包 ━━
── 1. 当前任务 ──    ← 本章要写什么
── 2. 主角状态 ──    ← 主角精确的当前状态（等级/属性/位置）
── 3. 出场角色 ──    ← 本章涉及的角色
── 4. 活跃伏笔 ──    ← 未回收的伏笔（超期标记）
── 5. 风格参考 ──    ← 最近3章节奏特征
── 6. 约束提醒 ⚠    ← 不能违反的硬约束
```

### 写作后

- 更新主角状态
- 更新伏笔登记表
- 更新事件索引
- 生成章节摘要
- Git commit

### 维护的文件

| 文件 | 内容 | 更新时机 | 读取者 |
|------|------|---------|--------|
| `状态/主角.md` | 等级/属性/位置/任务/装备 | 每章完成后 | 写作/规划/审稿 Agent |
| `状态/角色/王磊.md` | 配角状态/关系/已知信息 | 该角色出场后 | 写作/审稿 Agent |
| `状态/伏笔登记表.md` | 编号/内容/埋下章/预计回收章 | 每章完成后 | 规划 Agent |
| `记忆/事件索引.md` | 重大事件按章索引 | 每章完成后 | 快速定位历史事件 |
| `记忆/章节摘要/第N章.md` | 200 字章节概要 | 每章完成后 | 上下文包素材 |
| `记忆/风格参考.md` | 最近 5 章写作特征 | 每章完成后 | 风格一致性保持 |

## 规划 Agent（构思者）

**职责**：剧情构思、大纲生成、设定扩展、反转设计

- **输入**：当前故事状态 + 未回收伏笔
- **输出**：3-5 个剧情方向 + 对比表
- **原则**：不出单一方案，至少给 3 个选择

典型场景：

```
用户："没灵感了，后面怎么写？"
规划 Agent：
  → 读取当前主角状态、未回收伏笔
  → 分析剧情张力点
  → 输出 3 个可选方向（含优缺点对比）
  → 用户选择后深化为章节大纲
```

## 写作 Agent（写手）

**职责**：按大纲和上下文包生成正文

- **输入**：上下文包 + 章节大纲
- **输出**：2000-3000 字正文
- **约束**：遵守项目模板 + human-linguistics 语感规则
- **原则**：不自己编剧情、不审查自己的稿子

### 知识支持

写作 Agent 可通过 RAG 层查询相关知识：

| 查询 | 获取的知识 |
|------|-----------|
| "这段太像 AI 写的" | 去AI味最小修改指南 |
| "写之前该准备什么" | 章节写前准备清单 |
| "这个大纲能展开吗" | 大纲质量评估清单 |

---

## 审稿 Agent（质检员）

**职责**：调用各审查子模块，产出统一审查报告

- **审查范围**：语感 / 设定 / 情节 / 节奏 / 数值
- **调用子模块**：game-datafied / human-linguistics / plot-review / rhythm-review
- **原则**：只挑毛病，不写替代方案

### 审查流程

```
审稿 Agent 收到正文
  │
  ├── human-linguistics  → 语感检查（总结腔、AI味、句式单一）
  ├── plot-review        → 情节检查（逻辑漏洞、伏笔回收）
  ├── rhythm-review      → 节奏检查（爽点密度、疲劳点）
  └── game-datafied      → 题材检查（数值合理性、成长曲线）
  │
  ▼
统一审查报告（分类列出问题，不替作者修改）
```

## Agent 协作流程

### 单章完整流水线

```
用户确认"写第N章"
  │
  ▼
① 上下文 Agent — 打包当前上下文
   ├─ 主角状态（等级/属性/位置）
   ├─ 出场角色
   ├─ 活跃伏笔（标记超期）
   ├─ 风格参考（最近3章特征）
   └─ 约束提醒（⚠ 不能违反的规则）
  │
  ▼
② 写作 Agent — 生成正文
   ├─ 接收上下文包 + 大纲
   ├─ 写 2000-3000 字
   ├─ RAG 查询：语感规则、写前清单
   └─ 遵守 human-linguistics 语感规则
  │
  ▼
③ 上下文 Agent — 写作后归档
   ├─ 更新主角状态
   ├─ 更新伏笔登记表
   ├─ 更新事件索引
   ├─ 生成章节摘要
   └─ git commit
  │
  ▼
④ 审稿 Agent — 质量检查
   ├─ 语感检查（human-linguistics）
   ├─ 情节检查（plot-review）
   ├─ 节奏检查（rhythm-review）
   └─ 题材检查（game-datafied）
  │
  ▼
⑤ 总编 Agent — 汇总呈现
   ├─ 展示修改建议
   └─ 等待用户确认
```
