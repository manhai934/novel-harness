# 审稿输出模板

> 主模块 Step 3 的详细输出模板。按需加载，仅在实际产出审稿报告时查阅。
> 根据用户等级选择对应模板。

---

## Lv.1 零基础小白 — 鼓励 + 对比 + 少问题

```
━━ 本次审稿总结 ━━

先说好的：
- {优点1}
- {优点2}
- {优点3}

建议改 2 个地方就够了：
1. [问题位置] {具体问题}
   ❌ 原文：{你的写法}
   ✅ 参考：{修改后的写法}
   为什么：{用一句话解释原因}

2. [问题位置] {具体问题}
   ❌ 原文：{你的写法}
   ✅ 参考：{修改后的写法}
   为什么：{用一句话解释原因}

下一步：先改这两个，改完发我，咱们再看剩下的。
```

---

## Lv.2 老书虫转作者 — 结构诊断 + 模板建议

```
━━ 本次审稿总结 ━━

框架问题：
- {结构性问题1}
- {结构性问题2}

细节问题（按优先级排列）：
1. [问题位置] {问题} — 规则依据：{规则出处}
2. [问题位置] {问题} — 规则依据：{规则出处}
3. [问题位置] {问题} — 规则依据：{规则出处}

参考写法：
{给一个示范段落，不用全改}

字数统计：{总字数}，建议调整到 {目标字数}
```

---

## Lv.3 资深作者 — 问题标记 + 由作者决策

```
━━ 本次审稿总结 ━━

标记问题（共 {N} 处）：
1. [第X段] {问题简述}
   → 你之前在这个位置的处理方式好像不太一样，看你怎么想
2. [第X段] {问题简述}
   → 这个设定和世界观.x.md有点出入，你看看是不是有意为之

备注：{补充观察，不做定论}

你怎么看这些点？需要我展开说还是你已经知道怎么改了？
```
