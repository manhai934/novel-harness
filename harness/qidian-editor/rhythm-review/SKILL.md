# rhythm-review · 节奏/爽点审查 skill

> 本模块是 `qidian-editor` 的**专项审查子模块**。
> 职责：分析章节的阅读体感——高潮间隔是否合理、钩子密度够不够、爽点分布是否均匀、章节长度是否稳定。

---

## 1. 触发条件

以下情况应加载本 skill：

- **常规触发**：写完一卷（5-10章）后，做一次整体节奏检查
- **数据预警**：追读率下降、读者评论反馈"水""无聊""太慢热"
- **上架前**：前 5-10 章的节奏是否足以留住读者
- **开篇检查**：前三章的钩子密度和爽点频率是否达标
- **用户指定**：用户说"帮我看看节奏""感觉最近写得太平了"

**不适用**：单章语病检查、设定一致性检查时不需要加载本 skill。

---

## 2. 执行流程

### 2.1 准备阶段

```
输入：审查范围（哪几章 / 全文）
加载：
  □ 目标章节的正文（或各章摘要）
  □ 各章字数统计
  □ 各章结尾句（检查钩子）
  □ 之前章节的节奏档案（如果有 changes/ 记录）
```

### 2.2 五维检查

按以下五个维度逐项分析，每项产出一个小结：

```
维度一：高潮间隔
  □ 标记每章的核心冲突/高潮点位置
  □ 计算相邻高潮之间的章节数
  □ ⚠ 连续 3 章无高潮 → 读者弃书预警
  □ ⚠ 高潮过于密集（每章都是高潮）→ 读者疲劳，爽点贬值
  ✅ 黄金节奏：小高潮 2-3 章一次，中高潮 5-7 章一次，大高潮一卷一次

维度二：钩子密度
  □ 每章结尾是否有悬念/钩子？
  □ 钩子类型是否丰富？（信息钩子/冲突钩子/情感钩子/悬念钩子）
  □ ⚠ 连续 3 章结尾平淡 → 读者不会点下一章
  □ ❌ 常见无效钩子："且听下回分解""预知后事如何"——AI 式结尾
  ✅ 有效钩子：新信息出现/冲突升级/角色陷入危机/关键对话中止

维度三：章节长度分布
  □ 计算每章字数，看标准差
  □ ⚠ 相邻章节字数差异超过 50% → 节奏断裂感
  □ ⚠ 连续 3 章字数递增/递减 → 节奏失控趋势
  ✅ 建议：正文章 2000-4000 字，章节间字数波动 < 30%

维度四：爽点类型分布
  □ 标记每章的核心爽点类型（见下方分类）
  □ 统计连续同类型爽点的次数
  □ ⚠ 连续 3 章都是战斗爽点 → 缺少升级/收获/日常缓冲
  □ ⚠ 连续 3 章都是日常/铺垫 → 读者会觉得"水"
  ✅ 黄金配比：战斗 40% + 收获 20% + 升级 20% + 日常/缓冲 20%

  爽点分类参考：
  ┌───────────┬────────────────────────────┐
  │ 类型      │ 示例                       │
  ├───────────┼────────────────────────────┤
  │ 战斗爽点  │ 打赢强敌、越级反杀、团灭   │
  │ 升级爽点  │ 等级提升、学习新技能       │
  │ 收获爽点  │ 获得装备、发现宝物、领奖励 │
  │ 装逼爽点  │ 打脸、震惊全场、身份揭露   │
  │ 情感爽点  │ 队友羁绊、拯救、牺牲       │
  │ 成长爽点  │ 心境突破、认知升级         │
  └───────────┴────────────────────────────┘

维度五：开篇吸引速度（仅限卷首/开书时检查）
  □ 第一章前 500 字是否抛出核心冲突或悬念？
  □ 前三章是否完成了"引出主角 → 建立目标 → 第一次冲突"？
  □ ⚠ 前三章没有明确冲突 → 读者不会继续看
  ✅ 黄金开篇：500 字内出冲突，2000 字内见系统/金手指，一章内第一次战斗
```

### 2.3 汇总

```
整理五维检查结果 →
  每个维度给出评分（A/B/C/D）
  标记风险点
  输出：rhythm_report.md
```

## 4. 质量门禁

| 门禁项 | 标准 | 结果 |
|:-------|:-----|:------|
| 连续无高潮 ≤ 3 章 | 不能超过 3 章没有高潮 | 通过 / 不通过 |
| 连续平淡结尾 ≤ 2 章 | 不能超过 2 章结尾无钩子 | 通过 / 不通过 |
| 章节字数波动 < 50% | 相邻章节字数差异不超过 50% | 通过 / 需关注 |
| 同类型爽点连续 ≤ 3 章 | 不能连续 3 章都是同类型爽点 | 通过 / 需关注 |
| 前三章首冲突 < 500 字 | 开篇必须在前 500 字内有冲突 | 通过 / 不通过 |

**任一"不通过" → 建议调整节奏后再进入下一环节。**

---

## 5. 回退路径

| 门禁结果 | 行动 |
|:---------|:------|
| **不通过（无高潮 > 3 章）** | 在最近的无高潮章节中插入小冲突/悬念 |
| **不通过（平淡结尾 > 2 章）** | 重写平淡章节的结尾，增加钩子 |
| **不通过（开篇吸引不足）** | 重写前 500 字，直接抛出冲突 |
| **需关注（字数波动大）** | 拆分或合并段落，使章节长度更均衡 |
| **需关注（同类型爽点连续）** | 在连续同类型爽点中插入不同类型的场景 |
| **通过** | 可进入下一环节（上架准备或归档） |
