# 用户身份档案

> 记录用户的写作经验等级，用于跨会话持久化身份识别。
> 每次确认身份后更新此档案，下次交互自动沿用。

---

## 字段说明

| 字段 | 说明 | 取值 |
|:-----|:------|:------|
| `level` | 经验等级 | `Lv1` (零基础) / `Lv2` (老书虫) / `Lv3` (资深作者) |
| `confirmed` | 是否经用户确认 | `true` (主动告知) / `false` (推断) |
| `genre` | 主要创作题材 | 如 `game-datafied` |
| `note` | 补充信息 | 用户自己说的特殊情况 |

## 使用方式

1. 确认身份后，更新 `profile.md` 中的字段
2. 每次加载主模块时，先读取此文件
3. 如果文件存在 → 沿用身份等级
4. 如果文件不存在或超过 30 天未更新 → 重新确认

## 更新触发条件

- 用户首次交互 → 创建档案
- 用户明确说自己是什么等级 → 更新确认状态
- 用户长时间未使用（>30天）→ 重置确认状态，重新问
