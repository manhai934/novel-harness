# 反馈追踪机制（自我改善）

> 当用户对 Agent 的输出不满意并指出问题时，记录下来。
> 同类问题出现 3 次 → 触发规则更新，永久改善。

---

## 核心理念

每次用户说"不对""不是这样""不符合我的场景"，都是学习机会：

```
用户发现不对 → 记录问题 → 同类问题 ×3 → 升级为规则 → 系统变聪明
```

**关键规则**：
- 不接受模糊否定（"这个不对"必须有原因）
- 记录的是"用户纠正了什么"，不是"用户不满意"
- 3 次不是硬数字 —— 如果某次用户特别强烈（"每次都犯这个"），可以直接提前触发

---

## 触发条件

以下情况记录一条反馈：

| 场景 | 触发条件 | 记录内容 |
|:-----|:---------|:---------|
| **审稿意见被拒** | 用户说"这个不用改""你理解错了" | Agent 的判断方向 vs 用户的真实需求 |
| **修改结果被回退** | 用户手动恢复了 Agent 做的修改 | 修改的具体内容 vs 用户恢复后的版本 |
| **创作灵感不匹配** | 用户说"不是我要的方向""这个不符合场景" | 创作方向 vs 用户的实际需求 |
| **设定扩展被否决** | 用户说"这个设定和世界观不符" | 扩展方案 vs 用户的设定边界 |
| **连续修正** | 同一个问题上用户纠正了 Agent 2 次以上 | 这个问题的完整链路 |

---

## 追踪流程

```
用户反馈"不对" →
  1. 记录这条反馈到 cases/feedback/
     → 日期 + 类型 + 详情

  2. 查询同类历史记录
     → 搜索 cases/feedback/ 中相同类型的条目
     → 如果同类记录已有 2 条 → 当前第 3 条

  3. 达到 3 次 → 触发规则更新
     → 生成 rule_proposal.md
     → 建议追加到哪里（语病诊断/语感词典/主模块规则）
     → 用户确认后追加

  4. 未达到 3 次
     → 记录在案，下次同类问题时会提示"同类问题已出现 N 次"
```

---

## 记录模板

每条反馈单独一个文件，按日期-类型编号。

**文件名格式**：`{YYYY-MM-DD}-{类型缩写}-{简短描述}.md`

| 类型缩写 | 含义 |
|:---------|:------|
| `review` | 审稿意见被拒 |
| `edit` | 修改结果被回退 |
| `idea` | 创作灵感不匹配 |
| `setting` | 设定扩展被否决 |
| `repeat` | 同一问题连续纠正 |

**模板内容**：

```markdown
# {YYYY-MM-DD} · 反馈记录

## 基本信息
- 类型：{review/edit/idea/setting}
- 来源：{审稿/灵感/设定扩展/修改}
- 用户场景描述：{用户当时在做什么}

## 原始输出
{Agent 输出的内容或修改建议}

## 用户反馈
{用户说的"不对""不是这样"等原话}

## 用户修正/期待的方向
{用户实际想要的，或手动修改后的版本}

## 差距分析
{Agent 的理解偏差在哪里：缺乏上下文？规则不明确？判断方向错误？}

## 同类历史
{搜索 feedback/ 的结果，当前第 N 次}
```
