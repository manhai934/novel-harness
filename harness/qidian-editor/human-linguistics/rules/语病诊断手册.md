# 语病诊断手册

> 13 类 AI 式高频语病，逐类诊断。
> 每类包含：❌ 病因 → ✅ 正确方向 → 示例对照。

---

### 2.1 过渡词过度使用

AI的标志性习惯：每换一个话题，必加一个过渡词。

| AI式过渡 | 人话替代 |
|:----------|:----------|
| 然而/然而/然而 | 直接接下一句，不需要过渡 |
| 不过 / 但是 | 但（一个字就够了） |
| 因此 / 因而 | 所以 / 于是 / 直接接 |
| 值得注意的是 | 删除，直接说事 |
| 与此同时 | 删除，换行即可 |
| 尽管如此 | 不过 / 但 / 删除 |
| 基于这个原因 | 所以 / 于是 |
| 在某种层面上 | 说起来 / 要说吧 |
| 换句话说 | 删除，直接重新说 |
| 总的来说 / 总而言之 | 删除 |

**门禁**：一段话里最多出现**一次**过渡词。连续两段都以"不过""但是""然而"开头 → 违规。

### 2.2 解释性旁白

AI最大毒点：解释"为什么"。角色做完一个动作，立刻补一句"因为他……"。

```
❌ "林北没有接话。因为他不想让话题继续下去。"
→ 删掉"因为"句。不接话就是不想接，不需要解释。

❌ "李思雨加快了脚步。她感觉到有人在后面跟着她。"
→ "李思雨加快了脚步。"就够了。读者知道为什么。

❌ "他决定先去副本看看。毕竟今天也没别的事可做。"
→ "他去了副本。" 或者 "反正没事，去副本看看。"
```

**门禁**：全文搜索"因为""毕竟""之所以"——每一处都问：这句话不说，读者能懂吗？

---

### 2.3 动作描写过度完整

AI的标准叙事公式：动作A → 连接 → 动作B → 目的。

```
❌ "他伸手从桌上拿起杯子，送到嘴边喝了一口。"
→ "他喝了口水。" 六个字能说清的事不要用二十个字。

❌ "她从口袋里掏出手机，点亮屏幕看了一眼时间。"
→ "她看了一眼手机。" 或 "她掏手机瞄了眼时间。"

❌ "他转过身，朝着门口的方向走了过去。"
→ "他往门口走去。"
```

**门禁**：不是每个动作都需要"起→承→转→合"。连续的日常动作可以直接跳到关键结果。

---

### 2.4 情感标签化

AI习惯用抽象的"情感词"给角色贴标签，而不是让读者从行为和细节中感受。

```
❌ "他感到一阵深深的恐惧。"
→ "他手心全是汗。" / "他腿有点发软。"

❌ "她心里充满了感动。"
→ "她鼻子一酸。" / "她别过头去不说话。"

❌ "林北非常惊讶。"
→ "林北愣了一下。" / "林北眉头皱了起来。"
```

**门禁**：能写"身体反应"就不要写"心理感受"。身体反应是 show，心理感受是 tell。

---

### 2.5 因果链过度完整

AI喜欢把"因为A所以B然后C"完整写出来。普通人会跳步。

```
❌ "因为魔化鸡的防御很低，所以林北一剑就能秒杀。"
→ "林北一剑就把魔化鸡给秒了。"（原因不重要，结果已经说明了）

❌ "由于野猪皮糙肉厚，普通攻击打上去只能造成少量伤害。"
→ "一剑砍上去，【-1】。"
```

**门禁**：如果结果已经说明了一切，删掉原因。读者自己能推断。

---

### 2.6 全方位观察综合征

AI喜欢在角色进入新环境时，做一次"全景扫描描述"。普通人不会。

```
❌ "房间里有一张床、一张桌子、一把椅子、一个衣柜。墙上挂着一幅画，窗台上放着一盆花。窗帘是蓝色的，地板是木质的。"
→ "房间不大，一张床一张桌，收拾得还算干净。"（挑2-3个特征够了）

❌ "街道两旁是各种各样的店铺，有卖早餐的、卖蔬菜的、卖日用品的。行人来来往往……"
→ "街两边全是小店铺，早餐摊的油烟味飘了半条街。"
```

**门禁**：环境描写只写**角色当前会注意到**的东西。急着赶路的人不会数街上有几家店。

---

### 2.7 每段必推进综合征

AI的每一个段落都在推进剧情。但人写小说的时候就是会有"闲着没事的段落"。

```
AI结构：进门→对话→冲突→解决→下一场

人写结构：进门→对话→...沉默了一会儿→看了看窗外→...接着聊→冲突→解决→散了
```

**门禁**：在紧张的段落之间，**插入一段什么也没发生的文字**。这是节奏的呼吸空间。

---

### 2.8 标准答案式结尾

AI的段落结尾喜欢"总结陈词"——把这一段的核心意思再用一句话说一遍。

```
❌ 段尾："这就是数据化世界的残酷之处。"
❌ 段尾："林北知道，这一天终于还是来了。"
❌ 段尾："不管怎么样，他都得走下去。"
```

**门禁**：段尾就是段尾，不需要"盖章确认"。正文说清楚了，读者不需要最后一句总结。

---

### 2.9 多余时间副词

AI倾向于在所有动词前加上表示"正在做"的副词。

```
❌ "他正在打开面板查看经验值。"
→ "他打开面板看了一眼经验值。"

❌ "她正准备开口说些什么。"
→ "她张了张嘴。" / "她刚想说话——"
```

**门禁**：中文的"正在""正准备""刚要"大部分时候多余。直接写动作本身即可。

---

### 2.10 身份重复标签

AI喜欢反复提醒读者"这个人的身份是什么"。

```
❌ "作为社区工作者的林北"
❌ "身为攻略者的王磊"
❌ "作为一名普通人的林北"
```

**门禁**：除非身份信息是当前句子的关键判定依据，否则不需要加"作为/身为/作为一名"。直接写名字。

---

### 2.11 重复强调缺失

AI不会用重复强调——它觉得重复是浪费字数。但人写东西恰恰靠重复制造节奏感。

```
❌ AI：他点了点头。          ✅ 人写：他点点头。
❌ AI：她微笑了一下。        ✅ 人写：她笑了笑。
❌ AI：周围一片漆黑。        ✅ 人写：黑乎乎的一片。
❌ AI：房间里很安静。        ✅ 人写：沉默。沉默。
❌ AI：他沉默了。            ✅ 人写：沉默了沉默。
```

**门禁**：一段正文里至少找到一处"重复强调"用法。如果全文没有一个词是重复的 → 违规。

---

### 2.12 AI指纹词污染

有些词在网文里几乎从不出现，但AI特别喜欢。

**高危指纹词（见到即删/替换）：**

| 指纹词 | 人话替代 | 级别 |
|:-------|:---------|:-----|
| 全方位 | 各方面/全部/哪都 | ★★★ |
| 凸显/彰显 | 显得/看得出来 | ★★★ |
| 加持 | 帮忙/辅助/加上 | ★★★ |
| 助力 | 帮忙/帮 | ★★★ |
| 赋能 | ⛔ 绝对禁用 | ★★★ |
| 迭代 | 升级/换/更新 | ★★☆ |
| 闭环 | 收尾/结束/完整 | ★★☆ |
| 复盘 | 回顾/想想/琢磨 | ★★☆ |
| 抓手 | ⛔ 绝对禁用 | ★★★ |
| 对齐 | 同步/对上/一致 | ★★☆ |
| 维度 | 方面/角度/层面 | ★★☆ |
| 底层逻辑 | ⛔ 绝对禁用 | ★★★ |
| 沉淀 | 积累/攒下来/留下 | ★★☆ |
| 心智 | 心里/脑子里/观念 | ★☆☆ |
| 破圈/出圈 | 火了/传开了 | ★☆☆ |
| 触达 | 联系/到/推送 | ★☆☆ |
| 洞见/洞察 | 见解/发现/注意到 | ★☆☆ |

**门禁**：正文出现"赋能""抓手""底层逻辑" → 直接扣毒点。出现其他指纹词 → 替换。

### 2.13 动词同质化（"进行病"）

AI喜欢用少量万能动词（进行、做、实施）替代大量具体动词。

```
❌ "进行搜索"     → ✅ "翻了一遍"/"四处翻找"
❌ "进行攻击"     → ✅ "冲上去砍"/"抬手就是一剑"
❌ "进行观察"     → ✅ "打量了一圈"/"瞄了几眼"
❌ "实施救援"     → ✅ "把人拉出来"/"去救"
❌ "做出回应"     → ✅ "回了一句"/"应了一声"
❌ "采取行动"     → ✅ "动了"/"出手了"
❌ "获得提升"     → ✅ "升级了"/"涨了"
❌ "产生怀疑"     → ✅ "觉得不对劲"/"起了疑心"
❌ "做出选择"     → ✅ "选了"/"挑了"
❌ "进行评估"     → ✅ "掂量了一下"/"默默算了笔账"
```

**门禁**：全文搜索"进行""实施""做出""采取"——每一处问：能不能直接换成一个具体动词？
