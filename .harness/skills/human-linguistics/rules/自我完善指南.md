# 自我完善操作指南

> 本模块是一个**活文档**。以下情况都应该触发更新。

---

## 新增规则到各手册

### 发现新语病

```
条件：读到一段AI输出，明确感觉到"这里不像人写的"
操作：确定属于哪类语病，追加到 语病诊断手册.md 对应分类
      如果现有分类都不匹配，追加到末尾并编号为新类
格式：遵循该文件已有的 ## 2.N 格式
```

### 发现新语感对照

```
条件：注意到AI偏好某个词/结构，而人更常用另一个
操作：定位到 语感对照词典.md 对应分类（3.x），新增一行到对照表
```

### 新案例入库

```
条件：积累了一个完整的"AI原文 → 人写改写"案例
操作：按日期编号，追加到 .harness/cases/ 目录
格式：见 cases/template.md
```

## 定期维护

| 频率 | 操作 |
|:-----|:------|
| **每月一次** | 通读全文，合并重复条目，删除过时或错误的条目 |
| **每季度一次** | 将高频出现的语病提升到对应手册靠前位置 |
| **每半年一次** | 评估是否有新的AI语病模式出现（随AI升级，旧模式可能消失，新模式可能出现） |

## 联动使用流程

```
Step 1: AI生成正文
Step 2: 正则规则预处理（标点统一、数字汉字化、删除AI模板段落、替换套话）
         → reference/参考_AI人性化正则规则.md
Step 3: human-linguistics 审稿（排查语病、对照词典、检查句式节奏）
         → rules/语病诊断手册.md + 语感对照词典.md + 句式节奏档案.md
Step 4: 润色提示词 + LLM（把字句、虚词、心理代偿等语义级润色）
         → reference/参考_小说润色提示词.md
Step 5: 人工终审
```

> 三个工具的分工：正则干"删"的活（删废话、删AI模板），语感模块干"查"的活（查语病、查语感倾向），润色提示词干"改"的活（语义改写）。
