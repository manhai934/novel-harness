# AI 内容人性化正则规则集

> human-linguistics 子模块的配套参考文件。
> 本文件用于对 AI 生成的网文正文做**确定性预处理**：清掉模型元话、统一标点、压掉高频 AI 套话、做少量低风险口语化。
> 它不是最终润色器。正则只适合处理“字面稳定、误伤可控”的问题，真正的语义、人设、节奏和题材口气仍交给 LLM 或人工终审。

---

## 一、定位边界

### 1.1 正则应该做什么

正则适合做三类事：

1. **格式统一**：引号、省略号、破折号、Markdown 残留。
2. **硬删除**：明显不是正文的 AI 元说明、模板化总结、提示词残留。
3. **低风险替换**：把少数高频书面套话改成更自然的表达。

### 1.2 正则不应该做什么

以下内容不要交给正则硬改：

- 人物情绪外化：如“他很紧张”改成具体动作。
- 动作节奏重写：如长句拆短、对话打断、场景补细节。
- 题材口气判断：如电竞、求生、古言、都市的语感差异。
- 大规模同义词替换：会破坏作者原本的语气。
- 专有名词、技能名、道具名、系统面板、数值面板的自动改写。

一句话原则：

> 正则负责“清垃圾”和“轻轻扶一下”，不要负责“重写成人话”。

## 二、推荐参数

| 参数 | 默认值 | 说明 |
|:-----|:------|:-----|
| hard_delete_rate | 1.00 | AI 元说明、Markdown 残留、明显非正文内容，命中即删 |
| ai_phrase_replace_rate | 0.45 | AI 套话替换概率。建议保守，避免全文同质化 |
| word_replace_rate | 0.15 | 通用同义词替换概率。只用于低风险连词和副词 |
| novel_oral_replace_rate | 0.25 | 小说口语替换概率。只在现代/口语叙事中启用 |
| system_panel_protect | true | 保护系统面板、数值、道具名、技能名 |
| genre_safe_mode | true | 默认不做题材强绑定替换 |

旧版高概率替换容易把文本改得油、碎、重复。推荐默认保守，需要明显口语化时再调高。

## 三、处理流程

### 3.1 预处理顺序

```text
原文
  -> 保护块识别（系统面板/代码块/表格/专有名词）
  -> 删除 AI 元说明
  -> 删除 Markdown 残留
  -> 删除或压缩模板化段落
  -> 标点规范化
  -> 低风险套话替换
  -> 数字策略处理
  -> 可选小说口语替换
  -> 交给 LLM 语义润色
```

### 3.2 后处理顺序

```text
LLM 润色后正文
  -> 标点规范化
  -> 删除新增 AI 元说明
  -> 检查系统面板是否被改坏
  -> 人工终审
```

## 四、保护规则

正则执行前，先识别并保护以下文本，避免被同义词、数字汉字化、标点替换误伤。

### 4.1 系统面板

```regex
^【.*?】$
```

适用示例：

```text
【等级：Lv.1】
【生命值：23/32】
【获得：铁剑（普通）】
```

处理原则：

- 不替换其中的数字、英文、符号。
- 不把 `Lv.1` 改成 `Lv。一`。
- 不把 `+40` 改成 `加四十`。
- 不把 `x2`、`×2`、`1.5 倍`强制汉字化。

### 4.2 代码块和表格

```regex
```[\s\S]*?```
^\|.*\|$
```

处理原则：

- 不做标点替换。
- 不做数字汉字化。
- 不做套话替换。

### 4.3 专有名词和题材词

以下类型只做保护，不做自动替换：

- 人名、地名、组织名。
- 技能名、道具名、副本名、系统名。
- 电竞地图、职业队、选手名。
- 求生题材中的资源名、频道名、庇护所名。

## 五、标点规范化

### 5.1 引号

| 来源字符 | 替换为 | 说明 |
|:--------|:------|:-----|
| `"` | `「」` | ASCII 直双引号按奇偶配对转换 |
| `“”` | `「」` | 中文弯引号转直角引号 |
| `‘’` | `『』` | 二级引号 |

注意：

- 已经在系统面板 `【】` 内的内容不处理。
- 英文缩写、路径、命令里的引号不处理。

### 5.2 省略号

| 规则 | 替换为 |
|:-----|:------|
| `\.{3,}` | `……` |
| `…+` | `……` |

### 5.3 破折号

| 规则 | 替换为 |
|:-----|:------|
| `--+` | `——` |
| `[–—]+` | `——` |

## 六、AI 元说明删除

以下内容不是正文，命中后整段删除。

### 6.1 写作任务说明

```regex
^我将按照您的.*$
^按照您的要求.*$
^根据您(?:提供|给出|上传)的.*$
^以下是我(?:根据|为您|按照).*$
^接下来(?:故事|剧情|内容)(?:可能|将|会).*$
^本次(?:写作|润色|改写)(?:重点|将|主要).*$
^下面是(?:修改后|优化后|润色后)的.*$
```

### 6.2 分析/总结残留

```regex
^故事梗概[:：]?.*$
^人物设定[:：]?.*$
^写作思路[:：]?.*$
^修改说明[:：]?.*$
^优化方向[:：]?.*$
^以下为正文[:：]?.*$
```

### 6.3 Markdown 残留

| 规则 | 说明 |
|:-----|:-----|
| `^#{1,6}\s+` | Markdown 标题 |
| `^\s*[-*]\s+\*\*[^*]+\*\*[:：]` | 列表项标题 |
| `^\s*(?:---+|\*\*\*+|===+)\s*$` | 分割线 |
| `^\s*>\s*` | 引用块标记，按需删除标记保留内容 |

## 七、模板化段落处理

这类句式不一定整段删除，先判断它是不是正文叙述。

### 7.1 高风险整段删除

适合删除的场景：段落明显是分析、总结、说明，不是角色视角下的正文。

```regex
^(首先|其次|最后)[，,].*
^(第一点|第二点|第三点)[，,].*
^(其一|其二|其三)[，,].*
^(总的来说|综上所述|归根结底)[，,].*
^从.*角度(?:来看|来说|而言)[，,].*
```

### 7.2 正文中谨慎替换

这些词如果出现在角色内心或旁白里，不要粗暴删除，只做低风险替换。

| 原表达 | 替换候选 |
|:------|:---------|
| 然而 | 但、不过 |
| 因此 | 所以 |
| 此外 | 另外、还有 |
| 与此同时 | 这时候、同一时间、另一边 |
| 事实上 | 说白了、其实 |
| 显而易见 | 明摆着、很明显 |
| 毫无疑问 | 肯定、不用想也知道 |
| 值得注意的是 | 注意、要留意的是 |

## 八、AI 废话短语

### 8.1 默认删除

这些词常用于说明文和模型回答，正文中多数可以直接删除。

```text
综上所述
总而言之
由此可见
不难发现
需要指出的是
值得一提的是
值得注意的是
从某种意义上说
在一定程度上
就目前来看
概括来说
归根结底
```

### 8.2 谨慎处理

这些词有时是角色口气的一部分，不建议一刀切删除。

```text
事实上
实际上
严格来说
换句话说
不可否认
毫无疑问
显而易见
不言而喻
```

推荐策略：

- 出现在说明性段落：删除或替换。
- 出现在角色对话：保留或轻换。
- 出现在反讽、吐槽、职业分析语境：保留。

## 九、低风险替换表

### 9.1 连词和副词

| 原词 | 推荐替换 | 不推荐替换 |
|:-----|:---------|:-----------|
| 然而 | 但、不过 | 可是说来 |
| 但是 | 但、不过 | 然而 |
| 因此 | 所以 | 故而、因而 |
| 如果 | 要是、如果 | 倘若、若是 |
| 因为 | 因为、是因为 | 缘于、鉴于 |
| 立即 | 马上、立刻 | 当下 |
| 突然 | 突然、猛地 | 冷不丁 |
| 逐渐 | 慢慢、渐渐 | 日渐 |

原则：

- 不为了替换而替换。
- 不把现代短篇硬改成方言或年代文口气。
- 不使用明显书面或公文词作为替换候选。

### 9.2 动词

| 原词 | 推荐替换 | 注意 |
|:-----|:---------|:-----|
| 进行 | 做、开始、继续 | “进行攻击”可改“攻击/动手/冲上去” |
| 实现 | 做到、完成 | 系统面板里不要改 |
| 导致 | 造成、弄得 | 正文可换，规则说明可保留 |
| 产生 | 冒出、出现 | 视语境 |
| 解决 | 处理、搞定 | “搞定”只适合现代口语 |
| 观察 | 看、打量 | “仔细观察”可改“盯着看了一会儿” |

## 十、小说口语替换

口语替换只在现代、都市、游戏、求生、电竞等口语叙事中启用。古言、严肃现实主义、出版腔默认关闭。

### 10.1 可用替换

| 原词 | 替换候选 | 使用边界 |
|:-----|:---------|:---------|
| 看 | 瞥、扫、盯着看 | 不建议全篇改成“瞧” |
| 看见/看到 | 看见、瞧见、扫见 | “瞧见”有口气，少量使用 |
| 漆黑 | 黑、黑乎乎、黑漆漆 | 恐怖/求生场景可用 |
| 寂静 | 安静、静得很、死寂 | “死寂”偏氛围，不要滥用 |
| 慢慢 | 慢慢、磨磨蹭蹭 | 后者有角色态度 |
| 然后 | 然后、跟着、接着 | 不要机械替换 |
| 立即/立刻 | 马上、立刻 | “当下”偏书面/古风 |
| 逃跑 | 跑、拔腿就跑、转身就跑 | 动作场景可用 |

### 10.2 禁止替换候选

以下候选会制造新的 AI 味或不合现代网文口气，默认不用：

```text
业已、故而、缘于、鉴于、施用、撑持、揽获、敲定、
大抵、约莫、思量、径直、最要紧的、症结、门槛、
全方位、系统性、多维度、底层逻辑、赋能、闭环、链路、
沉淀、赛道、迭代、抓手、加持、助力、破圈、触达
```

### 10.3 不再推荐的旧规则

以下旧规则容易把文本改油，默认废弃：

| 旧规则 | 问题 |
|:------|:-----|
| `深吸一口气` → `深深吸了一口气` | 没有去 AI 味，只是加重套话 |
| `看了很久` → `看了很久很久` | 容易幼稚化、重复化 |
| `等了很久` → `已经等了很久很久了` | 语气拖沓，不能通用 |
| `第二天` → `到了第二天` | 多数场景只是废字 |
| `和/与/及` → `以及` | 会让正文更书面 |

## 十一、数字处理策略

数字不要无脑汉字化。网文里数字经常承担系统、等级、数值、时间和爽点反馈。

### 11.1 保留阿拉伯数字

以下场景默认保留：

- 系统面板：`Lv.1`、`23/32`、`+40`、`1.5 倍`。
- 道具数量：`木材 x2`、`晶币×15`。
- 游戏等级、技能等级、伤害数值。
- 现代金额、时间、楼层、房号、编号。

### 11.2 可汉字化

以下场景可以汉字化：

- 年份：`1987年` -> `一九八七年`。
- 年代：`80年代` -> `八十年代`。
- 叙述性小数字：`3个人` -> `三个人`，但系统语境保留。
- 日期：`5月21日` -> `五月二十一日`，现代新闻/日志体可保留数字。

判断原则：

> 数字是“读者读给自己听”的，可以汉字化；数字是“系统显示给角色看”的，保留。

## 十二、题材安全模式

### 12.1 求生题材

保护这些词：

```text
木材、石头、食物、水、庇护所、避难所、安全屋、世界频道、
交易频道、区域频道、晶石、晶币、耐久、血量、等级、掉落
```

建议：

- 不把资源名改得文艺。
- 不把系统提示改成长句。
- 不删除“重复确认”“看面板”等行为，它们是题材的一部分。

### 12.2 电竞题材

保护这些词：

```text
地图名、点位名、枪械名、战队名、选手名、平台分段、训练赛、天梯、rank
```

建议：

- 不把玩家吐槽改成标准书面语。
- 不把圈层黑话替换成通用解释。

## 十三、推荐实现伪代码

```text
function humanize_regex(text):
  protected = collect_protected_blocks(text)
  text = replace_blocks_with_tokens(text, protected)

  text = delete_ai_meta_paragraphs(text)
  text = remove_markdown_residue(text)
  text = normalize_punctuation(text)
  text = delete_hard_ai_phrases(text)
  text = replace_soft_ai_phrases(text, rate=0.45)
  text = replace_low_risk_words(text, rate=0.15)
  text = optional_oral_replace(text, rate=0.25, genre_safe=true)
  text = process_numbers(text, protect_system_panel=true)

  text = restore_protected_blocks(text, protected)
  return text
```

## 十四、质检清单

正则处理后，至少检查这几项：

- 系统面板有没有被改坏。
- 人名、地名、技能名、道具名有没有被替换。
- 是否出现过度口语化，比如全篇都是“瞧”“搞定”“冷不丁”。
- 是否把角色对话里的口癖删掉了。
- 是否把题材关键词改没了。
- 是否把原本有用的数字反馈改成难读的汉字。

如果命中以上问题，降低替换概率，或把对应区域加入保护块。

## 十五、与 LLM 润色的分工

| 问题 | 正则处理 | LLM/人工处理 |
|:-----|:---------|:-------------|
| AI 元说明 | 删除 | 不需要 |
| Markdown 残留 | 删除/清理 | 不需要 |
| 标点混乱 | 统一 | 不需要 |
| 高频套话 | 低风险替换/删除 | 复查是否影响口气 |
| 情绪太直白 | 不处理 | 外化为动作 |
| 对话太整齐 | 不处理 | 打断、加口癖、删解释 |
| 节奏太平 | 不处理 | 长短句调整 |
| 题材不真实 | 不处理 | 加题材细节和成本 |

最终原则：

> 正则只做第一刀。真正的人味，来自角色的身体反应、具体损失、口气毛边和题材细节。
