# novel-harness · 普通人语文语义参考（子模块）

> 本模块是 `novel-harness` 的**子模块**，专为降低AI式语感、还原普通人自然写作风格而设计。
> ⚠ **本文件为 L1 索引层**——只包含核心原则和规则目录。
> 具体规则分布在 `rules/` 目录下，按需加载。
>
> **核心定位**：不是语法纠错，而是语感对齐。让AI学会"普通人怎么写"，而不是"AI认为该怎么写"。

---

## ★ 使用说明

**调用方式**：配合根目录 `SKILL.md` 一起加载。

**上下文加载策略（四层架构）**：

```
L1（常驻）：本文件（~60行）—— 核心原则 + 规则索引
L2（按需）：rules/ 目录下的规则文件
L3（按需）：references/ 目录下的引用资料、提示词和正则规则
L4（按需）：cases/ 目录下的实战案例
```

**Agent 查规则流程**：
```
Agent 接到"查语病"
  1. 加载本文件（L1）→ 看到索引
  2. 按任务类型加载对应 rules/ 文件（L2）
  3. 如果需要深度参考，加载 references/ 下的引用文件（L3）
  4. 如果不够，查看 cases/ 找类似案例（L4）
```

## 一、语感核心原则（L1 常驻）

这些是判断"像不像人写的"的根本准则。所有具体规则都从这些原则派生。

### 原则 1：人写东西是有"废话"的

AI追求信息密度最大化，人不会。人会重复自己刚说过的意思、加一些不推进剧情的观察、跑题几秒钟再拉回来。**这不是缺点，这是"人味"的来源。**

```
AI式（每句都有用）：林北推开门，客厅里没有人。桌上放着一杯凉了的茶。他走到窗前拉开窗帘，外面的阳光刺眼。

人写（有废话版本）：林北推开门，客厅里没人。嗯，真没人。桌上那杯茶早就凉透了——也不知道放了多久了。他走到窗前把窗帘拉开，外头的阳光一下子涌进来，眼睛都给晃了一下。
```

### 原则 2：人不做多余的数学

AI喜欢精确计算——几剑、几分钟、多少经验、还剩多少。普通人作者靠"感觉"和"对比"。

```
AI式：野猪的血量从200掉到了137，还剩63点。按照每剑12点伤害计算，还需要6剑。
人写：野猪的血条已经下去一大截了。再来几剑的事。
```

### 原则 3：人写东西有"口气"

AI的叙述是平的，每个句子保持同样的距离感。人会随情绪起伏：紧张时句子变短，放松时句子变长，烦躁时用词变粗。

**叙述的语气应该跟着角色的状态走，而不是一直保持"标准叙事腔"。**

### 原则 4：人留给读者猜的空间

AI会把所有事情都解释清楚。普通人会选择"不说"，让读者自己去想。

```
AI式：林北没有回答，因为他不想让队友知道自己已经升级了。他怕引起不必要的麻烦。
人写：林北没有回答。
```

> **检查方法**：先问自己：**这句话像人写的吗？** 如果答案是否定的，看看违反了哪条原则。
