# 公共灵感整理与组合

> 适用于跨题材的灵感整理，不绑定具体题材。
> 目标不是直接定稿，而是把零散想法压缩成可判断、可比较、可继续推进的候选方向。

---

## 一、灵感整理四步法

### 1. 先判断灵感类型

把用户给出的内容先归到一类，不要一上来就硬扩写：

- 题材灵感
- 人设灵感
- 世界观灵感
- 情节钩子
- 金手指/能力灵感
- 情绪/体验灵感
- 已接近完整故事雏形

如果只是单一元素，不要假装它已经是完整故事。

### 2. 提炼一句话核心假设

统一压缩成：

```text
如果……会怎样？
```

这一步的作用是把散乱灵感变成可判断的故事问题。

### 3. 提炼情感内核

每个灵感都要回答：

- 读者主要满足点是什么？
- 这个点更偏爽、更偏痛、更偏治愈、更偏悬念，还是更偏猎奇？
- 读者为什么会想继续追？

### 4. 先看风险，再看扩展

灵感不是只看“有趣”，还要看：

- 同质化风险高不高
- 创作难度是否超出当前项目阶段
- 是否能支撑中长篇，而不是只够一个开头
- 是否和作者当前能力匹配

---

## 二、公共评估维度

不必打复杂分数，但至少逐项判断：

| 维度 | 要问什么 |
|:-----|:---------|
| 核心概念 | 这一点子一句话能不能说清 |
| 情感内核 | 读者到底爽什么 / 想看什么 |
| 冲突潜力 | 能不能不断制造矛盾与推进 |
| 扩展空间 | 能撑几章 / 几卷 / 多长 |
| 创作难度 | 需要多复杂的设定、人物、逻辑 |
| 项目匹配 | 和当前项目、作者能力是否匹配 |
| 风险提示 | 最容易崩在哪、俗在哪、拖在哪 |

---

## 三、组合创新规则

### 1. A+B 组合

先用两元素组合，不要一开始就四五个元素乱堆。

优先组合：

- 题材 + 卖点
- 世界观 + 人设钩子
- 冲突类型 + 情绪体验

### 2. A+B+C 组合

第三个元素只用来做差异化，不要改变主轴。

建议顺序：

```text
主轴题材
+ 核心卖点
+ 一个差异化包装
```

### 3. 组合时的红线

- 元素越多，不等于越新。
- 只要出现两个以上强主轴，就要警惕失焦。
- 创新点应该增加记忆点，不应该增加理解成本。

---

## 四、输出时至少包含的字段

每个候选方向至少说清：

```text
一句话假设：
灵感类型：
情感内核：
核心冲突：
初步爽点：
扩展空间：
风险提示：
推荐优先级：
```

---

## 五、什么时候可以继续推进

符合以下条件再进入大纲或章节规划：

- 一句话假设清楚
- 情感内核不空
- 至少能看见一个中期发展方向
- 风险已经提前说出来

如果这四项答不清，先继续整理灵感，不要直接写正文。
