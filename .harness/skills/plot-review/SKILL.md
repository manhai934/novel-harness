# plot-review · 情节一致性审查 skill

> 本模块是 `novel-harness` 的**专项审查子模块**。
> 职责：检查故事内部的逻辑一致性——角色是否OOC、时间线是否对齐、伏笔是否回收、设定是否吃书。

---

## 1. 触发条件

以下情况应加载本 skill：

- **常规触发**：写完 3-5 章后，做一次整体一致性扫描
- **章节修改后**：对某章做过大幅修改（删角色/改剧情/调顺序），检查是否有残留引用
- **新增设定后**：世界观有新设定加入，检查旧章节是否需要联动更新
- **上架前**：全卷一致性通扫
- **用户指定**：用户说"帮我查查有没有前后矛盾"或"检查一下逻辑"

**不适用**：单纯查语病或润色时不需要加载本 skill。

---

## 2. 执行流程

### 2.1 准备阶段

```
输入：审稿范围（哪几章 / 全文）
加载：
  □ 目标章节的正文
  □ 设定文件（世界观/角色档案/数值体系）
  □ 前序章节的摘要（如果有 changes/ 记录）
  □ 已有的错误案例（cases/ 中相关条目）
```

### 2.2 六维检查

按以下六个维度逐项扫描，每项产出一个小结：

```
维度一：角色行为一致性
  □ 角色的性格与行为是否匹配？
  □ 角色的能力水平是否前后一致？
  □ 角色关系是否随剧情合理发展？（从陌生→熟人→朋友不能跳级）
  □ ⚠ 孤证不立——角色不能"突然变了一个人"而没有动机支撑
  
  *判定依据 → 加载 rules/角色关系金字塔.md*

维度二：时间线对齐
  □ 章节内的时间逻辑是否自洽？
  □ 跨章节的时间先后是否一致？
  □ 副本内时间流速 vs 外界时间是否一致？
  □ ❌ 常见违规："早上进的副本，出来已经天黑了"却没有交代时间流速差异

维度三：伏笔回收跟踪
  □ 已埋下的伏笔预计在多少章内回收？
  □ 超期未回收的伏笔有哪些？（超过设定篇幅的 50% 未回收 → 提醒）
  □ 已回收的伏笔是否有明确标记？
  □ ❌ 常见违规：第三章提到"项链有点奇怪"，二十章了还没下文

维度四：信息边界检查（与主模块 3.10 联动）
  □ 角色是否知道了不该知道的信息？
  □ 旁白是否替角色预判了未发生的事情？
  □ 刚认识的人是否互相预判了性格？
  
  *判定依据 → 加载 rules/角色知识边界.md*

维度五：数值体系一致性
  □ 同一条件下的攻击/防御/经验数值是否一致？
  □ 装备掉落是否匹配怪物等级？
  □ 升级所需经验曲线是否平滑？
  □ ❌ 常见违规：同一把剑上一章砍 50，这一章砍 80，没有交代变化原因

维度六：已删剧情残留（与主模块 3.9 联动）
  □ 被删除的角色名是否在后续章节还有引用？
  □ 被删除的情节是否在其他地方还有关联描述？
  □ 全文搜索被删内容的关键词
```

### 2.3 汇总

```
整理六维检查结果 →
  按严重程度分级：
    ★ 致命矛盾（必须修改）
    ★★ 需要关注（建议修改）
    ★☆☆ 小瑕疵（可选修改）
  输出：consistency_report.md
```

## 4. 质量门禁

| 门禁项 | 标准 | 结果 |
|:-------|:-----|:------|
| 致命矛盾 = 0 | 没有必须修复的逻辑漏洞 | 通过 / 不通过 |
| 伏笔超期率 < 20% | 超期未回收伏笔不超过总量的 20% | 通过 / 需关注 |
| 信息边界违规 = 0 | 角色不得知道不该知道的信息 | 通过 / 不通过 |
| 残留引用 = 0 | 已删内容没有残留引用 | 通过 / 不通过 |

**任一"不通过" → 必须修改后才能进入下一环节。**

---

## 5. 回退路径

| 门禁结果 | 行动 |
|:---------|:------|
| **不通过（致命矛盾 > 0）** | 列出所有致命矛盾，要求逐一修改后重新审查 |
| **不通过（信息边界违规）** | 标记违规位置，修改后重新审查该维度 |
| **不通过（残留引用）** | 全文搜索并删除残留引用 |
| **需关注（伏笔超期）** | 建议在后续章节安排回收，不影响当前发布 |
| **通过** | 可进入下一环节（节奏审查或上架准备） |
