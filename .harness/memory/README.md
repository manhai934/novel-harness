# 记忆系统

> 面向 200 万字长篇创作的文件化记忆基础设施。
> 所有状态文件由上下文 Agent 维护，其他 Agent 只读。
> 原则：**一次写入，到处读取，只有一份真相。**

---

## 目录结构（项目级，位于 `projects/{项目}/` 下）

```
projects/{项目}/
├── 正文/                       ← 已完成的章节
│   ├── 第1章.md
│   └── 第2章.md
├── 大纲/                       ← 创作蓝图
│   └── 第N章.md
├── 状态/                       ← ★ 上下文 Agent 维护
│   ├── 主角.md                  ← 主角状态（每次写作后更新）
│   ├── 角色/                    ← 其他角色状态
│   └── 伏笔登记表.md            ← 伏笔生命周期管理
└── 记忆/                       ← ★ 上下文 Agent 维护
    ├── 事件索引.md               ← 重大事件按章节索引
    ├── 章节摘要/                 ← 每章 200 字摘要
    │   ├── 第1章.md
    │   └── 第2章.md
    ├── 风格参考.md               ← 最近5章风格特征
    └── 上下文包日志/             ← 历史上下文包（可选，用于调试）
```

## 写入者 vs 读取者

| 文件 | 写 | 读 |
|:-----|:----|:----|
| `状态/主角.md` | 上下文 Agent | 写作/规划/审稿 Agent |
| `状态/角色/*.md` | 上下文 Agent | 写作/审稿 Agent |
| `状态/伏笔登记表.md` | 上下文 Agent | 规划 Agent |
| `记忆/事件索引.md` | 上下文 Agent | 规划 Agent |
| `记忆/章节摘要/*.md` | 上下文 Agent | 上下文 Agent（下次打包时读） |
| `记忆/风格参考.md` | 上下文 Agent | 写作 Agent |
| `大纲/*.md` | 规划 Agent + 用户 | 上下文/写作 Agent |
| `正文/*.md` | 写作 Agent | 上下文/审稿 Agent |

## 容量估算

| 项目 | 200 万字时 |
|:-----|:-----------|
| 章节数 | ~670 章（按 3000 字/章） |
| 章节摘要总量 | ~13 万字（200 字 × 670） |
| 伏笔登记表 | ~200-500 行 |
| 事件索引 | ~500-1000 行 |
| 角色状态 | ~10-30 个文件 |
| 状态文件单次读取时间 | <1 秒（纯文件读前几行） |
