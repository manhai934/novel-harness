# Subagent 运行时与通用挂起恢复规则

> 用途：定义总编 Agent、专业 subagent、临时任务、工具调用、用户确认等待之间的通用协作边界。
> 核心原则：任何插队任务都不能吃掉原任务；挂起前保存恢复点，完成后回到原上下文继续执行。

---

## 1. 运行模型

novel-harness 的 Agent 流程按“主控 + 子任务 + 可挂起临时任务”运行：

```text
用户请求
  -> 总编 Agent 保存主任务上下文
  -> 总编 Agent 判断目标 subagent
  -> 如遇临时任务，挂起主任务
  -> 执行临时任务：用户确认 / subagent 查询 / MCP 工具调用 / RAG 检索 / 知识包安装
  -> 把确认后的结果注入任务上下文
  -> 恢复主任务
  -> 调用目标 subagent 执行
  -> subagent 返回结果
  -> 总编汇总并决定是否写入项目记忆
```

这里的 subagent 指：

- 上下文 Agent
- 规划 Agent
- 写作 Agent
- 审稿 Agent
- 临时素材推荐 subagent：只在开书、选题、知识包推荐等插队任务中短暂存在

subagent 不是新的全局入口，而是总编 Agent 派发出去的受控任务单元。

临时素材推荐 subagent 不需要成为新的项目入口，也不直接写入项目文件。它只负责检索题材、平台、知识包和素材候选，并把候选结果返回给总编 Agent。

---

## 2. 通用挂起触发场景

以下情况都可以触发挂起恢复机制，不局限于知识包：

- 需要用户确认选项：题材包、剧情方向、角色设定、章节路线、修改范围。
- 需要临时调用 subagent：让上下文 Agent 打包状态，让审稿 Agent 先查问题，让规划 Agent 补章纲。
- 需要调用工具：MCP 下载、RAG 重建、脚本扫描、文件索引、项目状态整理。
- 需要补上下文：当前项目不明确、章节目标缺失、角色状态不完整。
- 需要外部资料准备：知识包推荐、RAG 检索、参考规则加载、平台风格查询。
- 需要中断当前输出：用户临时插入新要求，但完成后仍要回到原写作/规划/审稿任务。

如果临时任务会改变原任务的输入、路径或判断依据，就必须先挂起原任务。

## 3. 三层上下文

为了避免知识污染，必须区分三类上下文：

| 层级 | 作用 | 是否持久化 | 示例 |
|:-----|:-----|:-----------|:-----|
| 项目上下文 | 当前小说的真实设定和长期状态 | 用户确认后才持久化 | 主角、世界观、能力体系、伏笔 |
| 任务上下文 | 本轮任务需要的临时信息 | 默认不持久化 | 本章目标、审稿范围、已确认参考知识包 |
| 查询上下文 | subagent/RAG 临时查到的材料 | 不持久化 | 某题材常见套路、平台风格、参考规则 |

硬规则：

- 查询上下文不能直接写入项目上下文。
- 任务上下文只能影响本轮规划、写作或审稿。
- 只有用户确认“写入设定/项目档案/长期规则”时，才能把任务结果升级为项目上下文。

## 4. Subagent 知识作用域

subagent 接收知识时必须带作用域标签：

```text
已确认参考知识包：
- {pack_id}：{用途说明}
- 作用域：本轮任务 / 当前章节 / 当前项目
- 匹配级别：精准匹配 / 相近参考
```

执行边界：

- 精准匹配：可以作为本轮任务的主要参考。
- 相近参考：只能借鉴结构、氛围、避坑点，不能直接套题材设定。
- 未确认知识包：不能启用。
- 已安装但未确认知识包：不能默认启用。
- subagent 自己查询到的新知识：只能作为“待确认建议”，不能直接混入正文或设定。

### 4.1 临时素材推荐 subagent 边界

当开书推荐、平台选择、知识包匹配会打断主流程时，总编 Agent 可以挂起主任务，并调用临时素材推荐 subagent。

允许它做：

- 检索本地 included、remote installed、server available 知识包。
- 根据用户方向筛选 2-4 个题材、平台或知识包候选。
- 标注每个候选的匹配级别：精准匹配 / 相近参考 / 平台口味参考。
- 返回推荐理由、风险和适用边界。

禁止它做：

- 替用户决定最终题材、平台或知识包。
- 创建项目档案、写入长期设定、修改 `.harness/current-project.md`。
- 生成大纲、黄金三章或正文。
- 把查询到的 RAG 内容直接升级为项目设定。
- 在用户确认前启用知识包。

临时素材推荐 subagent 的输出必须回传为：

```text
素材推荐 subagent 结果：
- candidate_options：{候选题材/平台/知识包列表}
- match_level：{精准匹配 / 相近参考 / 平台口味参考}
- risks：{误用风险或题材污染点}
- recommended_next_action：{等待用户选择 / 换一批 / 跳过素材 / 进入开书信息补全}
- scope：本轮任务
```

总编 Agent 接收结果后，只能把“用户确认过的选项”写入任务上下文：

```text
accepted_result：
- selected_topic：{用户确认题材}
- selected_platform：{用户确认平台}
- selected_pack：{用户确认知识包}
- next_action：{恢复后的下一步}
```

## 5. 通用挂起与恢复协议

知识包推荐、MCP 下载、RAG 重建、用户确认、临时 subagent 调用、脚本扫描都属于“可挂起临时任务”，不能吃掉原任务。

挂起前必须保存：

```text
任务恢复点：
- resume_id：本次挂起任务的临时编号
- original_request：用户原始请求
- target_agent：目标 subagent
- suspended_step：挂起前步骤
- collected_context：已确认上下文
- pending_action：当前插队的临时任务
- pending_options：等待用户确认的选项，没有则为空
- next_action：恢复后继续执行的动作
```

临时任务完成后必须产出：

```text
临时任务结果：
- result_type：用户确认 / subagent 输出 / 工具结果 / RAG 结果 / 知识包结果
- accepted_result：用户已确认可进入任务上下文的内容
- rejected_result：用户拒绝或仅供参考的内容
- scope：本轮任务 / 当前章节 / 当前项目
- resume_id：对应的恢复点编号
```

恢复后必须做到：

- 回到 `next_action`，继续原任务。
- 不重新询问已确认信息。
- 不重复执行已经完成的临时任务。
- 不擅自切换目标 subagent。
- 把用户确认后的结果传给目标 subagent。

## 6. 嵌套挂起

subagent 内部也可能遇到新的临时需求，例如写作 Agent 发现角色状态缺失，需要上下文 Agent 先补状态；审稿 Agent 发现需要额外规则，需要总编推荐知识包。

嵌套挂起必须遵守栈式恢复：

```text
主任务 A
  -> 挂起 A，执行临时任务 B
    -> B 内部又需要临时任务 C
    -> 挂起 B，执行 C
    -> C 完成，恢复 B
  -> B 完成，恢复 A
```

禁止跳层恢复：

- C 完成后不能直接改写 A 的项目设定。
- B 未完成前不能假装 A 已恢复。
- 每一层都必须保留自己的 `resume_id` 和 `next_action`。

## 7. 防知识污染门禁

以下行为禁止：

- 因为本地安装了某个题材包，就默认影响所有小说。
- 把相近知识包当成精准题材包使用。
- 把 RAG 检索到的参考案例改写成项目设定。
- 把审稿知识包里的判断标准永久写入当前项目，除非用户确认。
- subagent 绕过总编，自行安装、启用或切换知识包。
- 恢复任务时丢掉原始目标，改成“根据知识包重新开书”。
- 临时任务完成后丢掉原始上下文，改走新任务。
- 嵌套挂起时跳过中间层，直接污染更外层任务。

允许行为：

- subagent 把缺口反馈给总编：“需要额外知识包/素材推荐”。
- 总编给用户提供候选知识包。
- 用户确认后，知识包进入本轮任务上下文。
- 本轮结束后，总编询问是否把关键结论写入项目档案。
- 临时 subagent 只返回结果，不直接修改主任务目标。

---

## 8. 写入项目档案的条件

只有以下情况可以把知识或结论写入项目长期上下文：

1. 用户明确确认“采用这个设定/规则/题材方向”。
2. 该内容已经出现在用户确认过的正文或大纲中。
3. 总编 Agent 明确标注写入位置，如角色状态、世界观设定、伏笔登记、章节摘要。

示例：

```text
本轮使用了 survival-topic 作为参考，但不会自动写入项目设定。
如果你确认采用“木屋庇护所 + 资源循环 + 天灾压迫”作为长期框架，我再写入项目档案。
```

## 9. Agent 协作话术

### 9.1 Subagent 请求临时任务

```text
当前任务缺少 {上下文/用户确认/题材参考/审稿规则/工具结果}。
请总编 Agent 挂起当前任务，先完成 {临时任务}；完成前我不会继续生成最终结果。
```

### 9.2 总编恢复任务

```text
临时任务已完成：
- 结果类型：{用户确认/subagent输出/工具结果/RAG结果/知识包结果}
- 已确认进入任务上下文：{accepted_result}
- 作用域：{本轮任务/当前章节/当前项目}

现在恢复刚才的任务：{next_action}
```

### 9.3 知识包专项恢复

```text
知识包已准备好：
- 已确认参考知识包：{pack_id}
- 匹配级别：{精准匹配/相近参考}
- 作用域：本轮任务

现在恢复刚才的任务：{next_action}
```

### 9.4 防污染提示

```text
这个知识包只作为本轮参考，不会写入项目长期设定。
如果你想把其中某个方向固定下来，我会单独让你确认。
```
