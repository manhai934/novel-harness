# 反馈追踪流程

> 主模块 Step 4 的详细支撑文件。按需加载，仅在用户对输出提出否定时查阅。

---

## 核心流程

```
用户反馈"不对" → 
  1. 理解偏差：确认用户不满意的具体内容
  2. 记录反馈：按 cases/feedback/template-反馈记录.md 格式记录
  3. 查询同类历史：搜索 feedback/ 是否有相同类型的问题
  4. 如果同类问题已达 3 次：
     → 自动生成规则更新提案
     → 告知用户："同类问题已出现 3 次，建议追加规则：{提案}，是否确认？"
  5. 如果同类问题不足 3 次：
     → 记录在案
     → 提醒用户："同类问题已记录 {N}/3 次"
```

## 触发标准

以下情况算一次有效反馈记录：

- 用户明确说"不对""不是这样""不符合场景"
- 用户手动回退了修改
- 用户指出了理解偏差

## 3 次触发后的动作

1. 生成 rule_proposal.md（规则更新提案）
2. 建议追加到对应的规则文件
3. 等待用户确认后执行更新

---

## 身份感知解读

| 用户等级 | 说"不对"可能意味着 | 应对方式 |
|:---------|:------------------|:---------|
| **Lv.1 零基础小白** | "我看不懂"/"太多了消化不了" | 简化建议，减少数量，更多对比示例 |
| **Lv.2 老书虫** | "我的直觉告诉我这样不对" | 多问"你觉得问题在哪"，帮他把直觉转化为具体规则 |
| **Lv.3 资深作者** | "我有更好的方案"/"你分析错了" | 直接讨论分歧，把作者的方案纳入规则提案 |
